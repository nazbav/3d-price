# Калькулятор стоимости 3D-печати (две версии)

Данный проект содержит **версии** калькулятора для расчёта себестоимости печати на 3D-принтере:

1. [Первая версия (упрощённая)](https://nazbav.github.io/3d-price/index1.html)  
2. [Вторая версия (расширенная)](https://nazbav.github.io/3d-price/index2.html)  
3. [Третья версия (расширенная)](https://nazbav.github.io/3d-price/index3.html)
4. [Четвертая версия (расширенная)](https://nazbav.github.io/3d-price/index.html)
   
Вторая версия включает более подробный функционал:
- Хранение нескольких принтеров и материалов
- Учёт дополнительных расходов
- Полная поддержка создания, редактирования и удаления (CRUD)
- Настройка налогов, доставки и прочих параметров
- Автоматическое сохранение данных в **LocalStorage**
- Импорт/экспорт настроек в формате JSON

Ниже рассмотрим вторую версию подробнее, а также опишем **формулы**, которые используются для расчётов в калькуляторе.

---

## Быстрый старт

1. Откройте [страницу калькулятора](https://nazbav.github.io/3d-price/index2.html).
2. Перейдите на вкладки **«Принтеры»**, **«Материалы»** и **«Доп. расходы»**, чтобы внести свои данные.
3. На вкладке **«Калькулятор»** введите:
   - Выбранный принтер
   - Выбранный материал
   - Время печати, вес изделия, ставку оператора, налог и т.д.
4. Нажмите **«Рассчитать»** и получите итоговую стоимость.

### Запуск локально

Если хотите доработать код или запустить его без GitHub Pages:

1. Склонируйте или скачайте репозиторий.
2. Откройте файл `index2.html` (или `index.html` для первой версии) любым браузером.
3. Все скрипты и стили уже подключены, дополнительных зависимостей (кроме Bootstrap CDN) нет.

## Основные параметры и данные

1. **Принтеры**  
   - Название (name)  
   - Стоимость (cost) – общая стоимость принтера, включая ремонт, запчасти и т.п.  
   - Часы окупаемости (hoursToRecoup) – за сколько часов печати принтер должен окупиться  
   - Мощность (power) – в Вт (ваттах)  
   - Стоимость обслуживания (maintCostHour) – в руб/час (например, замена/чистка и т.д.)

2. **Материалы**  
   - Тип материала (name)  
   - Стоимость за 1 кг (costPerKg)  
   - Количество в наличии (availableKg)  
   - Процент отхода (wastePercent) – брак, огрызки при обрезании, поддержки и т.д.

3. **Доп. расходы**  
   - Описание (description)  
   - Стоимость (cost) – разовые траты (например, упаковка, контейнеры для хранения, фильтры), распределяемые пропорционально часам печати.

4. **Настройки калькулятора (calcSettings)**  
   - Ставка оператора (operatorRate), руб/час  
   - Рабочих часов в день (workHoursDay)  
   - Дата начала (startDate) – для расчёта примерной даты окончания  
   - Цена за 1 кВт⋅ч (costKwh)  
   - Доставка/Упаковка (shipping) – фиксированная сумма  
   - Налог (taxPercent) – % налога

## Формулы расчёта

Ниже представлены **основные формулы**, используемые при нажатии **«Рассчитать»** во второй версии калькулятора. Все значения берутся из введённых пользователем данных.

### 1. Амортизация принтера

Каждый принтер имеет параметр «Часы окупаемости» (`hoursToRecoup`). Если он равен \(H\), а стоимость принтера \(C\), то **стоимость за 1 час**:

```
costPerHourPrinter = C / H
```

Далее, умножаем на фактические часы печати (значение `calcHours`), чтобы получить вклад принтера в себестоимость:

```
costPrinterPart = costPerHourPrinter * hours
```

**Пример**:  
- Стоимость принтера: 30 000 ₽  
- Часы окупаемости: 600 ч  
- Фактические часы печати: 10 ч  

Тогда:  
- `costPerHourPrinter = 30000 / 600 = 50 ₽/ч`
- `costPrinterPart = 50 * 10 = 500 ₽`

### 2. Оплата оператора

Если оператор получает \(R\) руб/час, а печать займёт \(h\) часов:

```
costOperatorPart = h * R
```

**Пример**:  
- Ставка оператора: 200 ₽/час  
- Время печати: 10 часов  

Тогда:  
- `costOperatorPart = 10 * 200 = 2000 ₽`

### 3. Расход материала

Материал учитывает **вес изделия** в граммах (\(g\)) и **процент отхода** (например, поддержка, браки).  
Пусть процент отхода \(wastePercent\) (в процентах), а стоимость 1 кг материала \(M\) (руб/кг).  

Сначала считаем фактический расход (учитывая отход):  
```
realGrams = grams * (1 + wastePercent / 100)
```

Переведём стоимость за 1 кг в стоимость за 1 грамм:  
```
costPerGram = costPerKg / 1000
```

Итоговые затраты на материал:  
```
costMaterialPart = realGrams * costPerGram
```

**Пример**:  
- Вес изделия: 100 г  
- Отход: 5 %  
- Стоимость материала: 800 ₽ за 1 кг  

Тогда:  
- `realGrams = 100 * (1 + 5/100) = 100 * 1.05 = 105 г`
- `costPerGram = 800 / 1000 = 0.8 ₽/г`
- `costMaterialPart = 105 * 0.8 = 84 ₽`

### 4. Доп. расходы (пропорционально часам)

Все дополнительные расходы (`additional`), внесённые во вкладке «Доп. расходы», суммируются:

```
sumAdditional = Σ(cost)
```

Однако эти расходы считаются «инвестиционными» и делятся на суммарные часы окупаемости принтера \(H\). Пусть \(\sum A\) – сумма доп. расходов, а \(h\) – время текущей печати:

```
costPerHourAdditional = sumAdditional / H
costAdditionalPart = costPerHourAdditional * h
```

**Пример**:  
- Сумма всех доп. расходов: 2000 ₽  
- Часы окупаемости принтера: 500 ч  
- Время печати: 10 ч  

Тогда:  
- `costPerHourAdditional = 2000 / 500 = 4 ₽/ч`
- `costAdditionalPart = 4 * 10 = 40 ₽`

### 5. Электричество

Пусть мощность принтера \(P\) (в ваттах), время печати \(h\), а цена за 1 кВт⋅ч \(kwhCost\). Тогда:

```
costElectric = (P / 1000) * h * kwhCost
```

**Пример**:  
- Мощность принтера: 300 Вт  
- Время печати: 10 ч  
- Тариф на электроэнергию: 6 ₽ за 1 кВт⋅ч  

Тогда:  
- `(300 / 1000) = 0.3 кВт`  
- `0.3 * 10 = 3 кВт⋅ч`  
- `3 * 6 = 18 ₽`  

Итого: `costElectric = 18 ₽`

### 6. Обслуживание принтера

В калькуляторе есть поле «стоимость обслуживания (руб/час)» (`maintCostHour`):

```
costMaintenance = maintCostHour * h
```

**Пример**:  
- `maintCostHour = 5 ₽/ч`  
- Время печати: `10 ч`  

Тогда:  
`costMaintenance = 5 * 10 = 50 ₽`

### 7. Упаковка / доставка

Это фиксированные расходы \(S\), которые вы вручную задаёте в калькуляторе (поле «Доставка/Упаковка, ₽»).

```
costShipping = S
```

**Пример**:  
- `S = 200 ₽` (вписали вручную)  

Тогда итогово: `costShipping = 200 ₽`

### 8. Суммарная стоимость без учёта налога

Собираем все предыдущие составляющие:

```
subTotal =
    costPrinterPart
  + costOperatorPart
  + costMaterialPart
  + costAdditionalPart
  + costElectric
  + costMaintenance
  + costShipping
```

### 9. Налог

В калькуляторе можно указать налог \(taxPercent\) в процентах (например, 6 % для УСН или самозанятых). Тогда:

```
costTax = subTotal * (taxPercent / 100)
```

### 10. Итоговая стоимость

```
total = subTotal + costTax
```

### 11. Расчёт времени (дней) и ориентировочной даты окончания

- «Рабочих часов в день» = \(whDay\)
- Фактическое время печати = \(h\)
- Тогда приблизительное число рабочих дней:

```
daysNeeded = ceil(h / whDay)
```

- Если указана «Дата начала» (`startDate`), то к этой дате прибавляем `daysNeeded`, и получаем **ориентировочную дату окончания**.

## Пример общего расчёта

Допустим, у нас есть следующие вводные:

- **Принтер**:  
  - `cost = 20 000₽`, `hoursToRecoup = 400`, `power = 300Вт`, `maintCostHour = 5₽/ч`
- **Материал**:  
  - `costPerKg = 800₽`, `wastePercent = 5%`
- **Доп. расходы**: всего 1 000 ₽ (на всю жизнь принтера), т.е. 1 000 ₽ / 400 ч = `2,5₽/ч`
- **Печать**: `hours = 10ч`, `grams = 120г`
- **Ставка оператора**: `operatorRate = 200₽/ч`
- **Тариф за электричество**: `6₽/кВт⋅ч`
- **Доставка/Упаковка**: `300₽`
- **Налог**: `6%`

**Шаги**:

1. Амортизация принтера:  
   ```
   costPerHourPrinter = 20 000 / 400 = 50 ₽/ч
   costPrinterPart = 50 * 10 = 500 ₽
   ```
2. Оператор:  
   ```
   costOperatorPart = 10 * 200 = 2 000 ₽
   ```
3. Материал с учётом отхода:  
   ```
   realGrams = 120 * (1 + 5/100) = 126 г
   costPerGram = 800 / 1000 = 0.8 ₽/г
   costMaterialPart = 126 * 0.8 = 100.8 ₽
   ```
4. Доп. расходы (пропорционально часам):  
   ```
   sumAdditional = 1 000 ₽
   costPerHourAdditional = 1 000 / 400 = 2.5 ₽/ч
   costAdditionalPart = 2.5 * 10 = 25 ₽
   ```
5. Электричество:  
   ```
   costElectric = (300 / 1000) * 10 * 6 = 0.3 * 10 * 6 = 18 ₽
   ```
6. Обслуживание принтера:  
   ```
   costMaintenance = 5 * 10 = 50 ₽
   ```
7. Упаковка/доставка:  
   ```
   costShipping = 300 ₽
   ```
8. Суммарно без налога:  
   ```
   subTotal = 500 + 2000 + 100.8 + 25 + 18 + 50 + 300
            = 2993.8 ₽
   ```
9. Налог 6 %:  
   ```
   costTax = 2993.8 * (6 / 100) = 179.63 ₽
   ```
10. Итого:  
    ```
    total = 2993.8 + 179.63 = 3173.43 ₽
    ```

## Импорт и экспорт данных

На вкладке «Импорт/Экспорт JSON» вы можете:

- **Выгрузить JSON** со всеми текущими данными (принтеры, материалы, доп. расходы, настройки).  
- **Импортировать JSON**, чтобы загрузить данные из резервной копии или поделиться настройками с коллегами.

**Важно**: при импорте текущие данные **перезаписываются**.

---

## Где хранятся данные?

Все внесённые изменения (принтеры, материалы, доп. расходы, настройки) автоматически сохраняются в **LocalStorage** вашего браузера.  
Это означает, что после перезагрузки страницы или закрытия браузера данные не пропадают (до тех пор, пока вы не очистите кэш браузера вручную или не импортируете пустой JSON).

## Скрипт постобработки для Orca Slicer

В каталоге репозитория находится скрипт `open_calc.py`. Его можно указать в
настройках Orca Slicer как постпроцессор. Скрипт разбирает сгенерированный
`gcode`-файл, извлекает название принтера, время печати, вес и тип филамента,
затем открывает браузер со ссылкой на калькулятор, автоматически подставляя все
параметры. Использование:

```bash
python open_calc.py path/to/file.gcode
```

После выполнения будет открыт URL вида:
`https://nazbav.github.io/3d-price/index.html?printer=...` с подставленными
значениями.
