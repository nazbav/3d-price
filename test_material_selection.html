Material Selection Implementation Test

✅ Implementation Complete

Successfully modified the setupGcodeDrop() function to prompt users for material selection when no material is automatically resolved.

📋 Problem Statement

When a G-code file is dropped onto a printer table and resolveMaterialIdByProfile returns an empty or falsy value, the system should prompt the user to select a material instead of silently adding the model with no material.

🔧 Changes Made

Before (Original Code):

for (const f of files) {
  const txt = await f.text();
  const parsed = await parseGcodeText(txt);
  const mid = await resolveMaterialIdByProfile(parsed.matProfile);
  addModelRow(printerId, {
    id: parsed.code,
    name: parsed.code,
    thumbnail: parsed.thumb,
    status: 'new',
    hours: parseTimeFromLink(parsed.timeStr),
    weight: parseFloat(parsed.weight) || 0,
    materialId: mid  // ❌ Could be empty/falsy
  });
}

After (Modified Code):

for (const f of files) {
  const txt = await f.text();
  const parsed = await parseGcodeText(txt);
  let mid = await resolveMaterialIdByProfile(parsed.matProfile);
  
  // If no material found, prompt user to select from all available materials
  if (!mid) {
    const materialOptions = appData.materials.map(m => ({
      value: String(m.id),
      label: `${m.name} (ID: ${m.id})`
    }));
    
    if (materialOptions.length > 0) {
      const result = await showMultiPrompt('Выберите материал', [{
        id: 'materialId',
        label: 'Материал',
        type: 'select',
        options: materialOptions
      }]);
      
      if (result && result.materialId) {
        mid = result.materialId;
      } else {
        // User cancelled, skip this file
        continue;
      }
    } else {
      alert('Нет доступных материалов. Добавьте материалы во вкладке "Материалы".');
      continue;
    }
  }
  
  addModelRow(printerId, {
    id: parsed.code,
    name: parsed.code,
    thumbnail: parsed.thumb,
    status: 'new',
    hours: parseTimeFromLink(parsed.timeStr),
    weight: parseFloat(parsed.weight) || 0,
    materialId: mid  // ✅ Always has a value now
  });
}

🎯 Key Features Implemented

  • Automatic detection: Checks if resolveMaterialIdByProfile returns empty/falsy value
  • Material selection dialog: Shows all available materials with ID and name using showMultiPrompt
  • User choice handling: Only proceeds if user selects a material, skips file if cancelled
  • Error handling: Shows helpful message if no materials are available in the system
  • Graceful fallback: Preserves existing behavior when materials are found automatically

🔄 User Experience Flow

  1. User drops G-code file onto printer table
  2. System parses G-code and extracts material profile
  3. System attempts automatic material resolution
  4. If no material found:
    • Modal dialog appears with "Выберите материал" title
    • Dropdown shows all materials as "Material Name (ID: X)"
    • User selects material and clicks OK → Model added with selected material
    • User clicks Cancel → File skipped, no model added
  5. If material found automatically: Model added immediately (no change in behavior)
